《资源回收管理系统开发设计书》

## 1. 项目概要

### 1.1 项目背景

当前公司已有用户网页登录入口，可以在登录/注册时采集用户的基本个人信息（姓名、电话、个人番号卡等），并希望在此基础上建设一套“资源回收后台交易管理系统”，用于：

- 统一管理回收用户的身份信息与车辆信息；
- 由后台运营人员**手动录入**用户的回收交易记录；
- 按姓名、手机号码、品类、时间（按月/年）等多维度快速检索用户历史交易记录；
- 为后续数据统计与业务分析打基础。

### 1.2 建设目标

1. 与现有网页登录系统对接，自动同步用户基本信息到后台交易管理系统。
2. 支持后台人员对用户交易记录进行新增、修改、查询和导出。
3. 支持按关键字（姓名、手机号）、回收品类、时间范围（按月/按年）进行组合筛选。
4. 数据存储采用关系型数据库，保证数据一致性与扩展性。

---

## 2. 系统范围

### 2.1 范围内

- 用户基本信息与车辆信息管理（从现有登录系统同步/导入为主）。
- 交易管理：录入、维护用户回收交易记录。
- 分类管理：回收品类信息维护。
- 查询统计：多条件查询、简单汇总（按月/年）。
- 权限管理：后台管理员账号及操作权限（至少区分“管理员/录入员/查看者”）。
- 日志审计：交易录入与修改操作的日志记录。

### 2.2 范围外（本期不做）

- 移动端 APP / 小程序开发。
- 用户自助查看/下载交易记录的前台功能（可作为后期扩展）。
- 自动称重设备、扫码秤等硬件对接。
- 高级报表分析（BI、多维度可视化仪表盘等）。

## 3. 功能需求说明

### 3.1 与现有网页登录系统对接

**说明：** 现有登录页面在用户注册/完善资料时，提交姓名、电话、个人番号卡等信息，本系统需要获取这些信息，并存入本系统数据库。

**功能点：**

1. **用户信息同步接口**
   - 现有登录系统在用户注册/更新资料时，调用本系统接口：
     - 新增用户：发送姓名、电话、个人番号卡、登录账号标识（如 user_id）、邮箱（如有）、地址等。
     - 更新用户：无论手机号或其他信息变化时同步更新。
   - 本系统根据登录系统的 user_id 作为主键或外键进行关联，避免重复。

2. **车辆信息录入**
   - 用户在现有系统中录入个人车辆信息（车牌号、车辆类型等）时，同样通过接口同步到本系统；
   - 若前台暂不支持车辆录入，则由后台管理员在本系统中为用户维护车辆信息。

### 3.2 用户管理（后台）

**目标：** 后台能查看和维护系统中已经登录/同步过的用户信息。

**主要功能：**

1. 用户列表
   - 按姓名、电话、个人番号卡、user_id 查询。
   - 支持分页显示。
2. 用户详情
   - 基本信息：姓名、电话、个人番号卡、邮箱、地址、创建时间、状态等。
   - 车辆信息列表：车牌号、车辆类型、备注。
   - 用户历史交易记录入口快捷链接。
3. 用户信息编辑
   - 为保持主数据一致性，用户基本信息以前台系统为主，本系统通常只读；
   - 可支持管理员添加备注字段（内部使用）。
4. 车辆信息管理
   - 新增车辆：车牌号、车辆类型、默认车辆标记等。
   - 编辑/删除车辆。

### 3.3 交易管理

由后台工作人员根据实际回收情况，**手动录入**用户的交易记录。

**主要功能：**

1. 新增交易记录
   - 选择用户：通过姓名/手机号/个人番号卡搜索用户并选择。
   - 交易基本信息：
     - 回收日期（默认当前日期，可修改）。
     - 回收品类：从“品类字典表”中选择。
     - 重量（kg 等单位，数字类型）。
     - 单价（可选，若业务需要）与交易金额：
       - 若输入单价与重量，可自动计算交易金额；
       - 也允许直接输入交易金额。
     - 车辆信息：可选择该用户名下车辆，或手填车牌号（临时车辆）。
     - 结算方式：现金 / 转账 / 其他（可选）。
     - 备注：自由文本。
2. 编辑/删除交易记录
   - 可编辑交易记录（重量、金额等字段）；
   - 删除需严格控制权限，并记录操作日志（谁在何时删除了哪条交易）。
3. 交易列表
   - 默认按交易日期倒序。
   - 支持按用户关键词、日期范围、品类、金额区间查询。
   - 支持导出为 CSV/Excel。

### 3.4 品类管理（Category）

**功能需求：**

1. 品类字典维护
   - 新增/编辑/停用回收品类：
     - 品类编号
     - 品类名称（如“废纸箱”“废铁”“塑料瓶”等）
     - 单位（kg、吨、件等）
     - 默认单价（可选）
     - 状态（启用/停用）
2. 品类查询
   - 按品类名称关键字搜索。
   - 按状态（启用/停用）过滤。

### 3.5 查询与统计功能

**目标：** 支持运营人员根据多种条件快速查询用户历史交易记录，并进行简单统计。

**查询条件示例：**

- 关键字查询：
  - 姓名（模糊匹配）
  - 手机号码（精准匹配或模糊）
  - 个人番号卡
- 回收品类：
  - 单一品类
  - 多品类选择（可选）
- 时间维度：
  - 指定日期范围
  - 按月统计（如 2025-01，2025-02）
  - 按年统计（如 2024 年全年的总量）
- 交易金额区间：
  - 最小金额 / 最大金额

**展示内容：**

1. 查询结果列表
   - 每条记录信息包括：
     - 用户姓名、手机号
     - 回收日期
     - 品类名称
     - 重量
     - 单价（如有）
     - 交易金额
     - 车辆信息
2. 汇总统计（简单版）
   - 当前查询条件下：
     - 总交易笔数
     - 总重量
     - 总金额
   - 如按月/年统计：
     - 每月/每年汇总金额及重量（类似报表）。

## 4. 非功能需求

### 4.1 性能

- 预计日交易录入量：约 XXX 笔/天（由业务方评估）。
- 查询响应时间：普通查询在 2 秒内返回结果（在 1 万级数据量情况下）。
- 支持至少 20–50 名后台用户同时在线操作。

### 4.2 安全性

- 后台登录需账号密码，并支持 2FA（如有需要）。
- 所有接口通过 HTTPS 访问。
- 对外提供的 API 做 IP 白名单及签名机制限制（与现有登录系统对接时）。
- 用户敏感信息（电话、个人番号卡）需加密存储或脱敏显示。

### 4.3 权限管理

- 角色建议：
  - 系统管理员：管理账号、权限、系统配置、品类维护。
  - 业务管理员：录入/编辑/删除交易记录，查看报表。
  - 只读查看者：仅可查询，不可修改。
- 不同角色访问不同菜单和操作。

### 4.4 日志与审计

- 登录日志：记录后台登录成功/失败、IP、时间。
- 操作日志：记录新增/修改/删除交易记录、用户车辆信息的操作。
- 查询日志（可选）：记录关键性的查询操作（比如导出大量数据）。

## 5. 系统架构设计

### 5.1 总体架构

建议采用典型的 B/S 架构（三层）：

- 表现层（Web 前端）
  - 提供后台管理界面，支持 PC 浏览器访问。
- 应用层（后端服务）
  - 实现业务逻辑：用户同步接口、交易录入、查询统计等。
- 数据层（关系型数据库）
  - 建立规范化的数据结构，保证数据一致性。

### 5.2 与现有系统的集成方式

- 通过 RESTful API 的形式提供用户信息同步接口：
  - 登录系统 → 本系统：
    - POST /api/users/sync
    - POST /api/users/{id}/vehicles/sync
- 授权方式：
  - 使用 Token 或签名参数（timestamp + app_key + sign）保证调用安全。

## 6. 数据库设计（概要）

以关系型数据库（如 MySQL/PostgreSQL）为例，设计核心数据表：

### 6.1 用户表（users）

- id (PK)
- external_user_id：前台登录系统的用户 ID（唯一）
- name：姓名
- phone：手机号
- personal_id：个人番号卡
- email：邮箱（可选）
- address：地址（可选）
- status：状态（正常/禁用）
- created_at
- updated_at

### 6.2 车辆表（user_vehicles）

- id (PK)
- user_id (FK → users.id)
- plate_number：车牌号
- vehicle_type：车辆类型（货车、小型车等）
- is_default：是否默认车辆
- remark：备注
- created_at
- updated_at

### 6.3 回收品类表（categories）

- id (PK)
- code：品类编码
- name：品类名称
- unit：计量单位（kg、吨、件）
- default_price：默认单价（可空）
- status：启用/停用
- created_at
- updated_at

### 6.4 交易表（transactions）

- id (PK)
- user_id (FK → users.id)
- vehicle_id (FK → user_vehicles.id，可空)
- category_id (FK → categories.id)
- trade_date：交易日期
- weight：重量
- unit_price：单价（可空）
- amount：交易金额
- payment_method：结算方式（枚举：cash/transfer/other）
- remark：备注
- created_by (FK → admin_users.id)：录入人
- created_at
- updated_at

### 6.5 管理员用户表（admin_users）

- id (PK)
- username
- password_hash
- role：角色（admin/operator/viewer）
- status
- last_login_at
- created_at
- updated_at

### 6.6 操作日志表（operation_logs）

- id (PK)
- admin_user_id
- action_type（如 create_transaction / update_transaction / delete_transaction）
- target_type（transaction/user/vehicle）
- target_id
- detail：JSON 字段，记录变更前后数据摘要
- created_at
- ip_address

## 7. 主要业务流程

### 7.1 用户注册 / 登录与信息同步流程

1. 用户在现有登录系统注册或首次登录，输入姓名、电话、个人番号卡等信息。
2. 登录系统校验并写入自身数据库。
3. 登录系统调用本系统接口 `/api/users/sync`：
   - 如果 external_user_id 不存在：在本系统创建新用户。
   - 如果存在：更新姓名、电话等信息。
4. 同步成功后，本系统数据库中即存在对应用户记录。

### 7.2 车辆信息录入流程

1. 用户在前台登录系统维护车辆信息（如有）。
2. 登录系统调用本系统 `/api/users/{id}/vehicles/sync` 接口。
3. 本系统在 `user_vehicles` 表中插入/更新车辆数据。

（如由后台维护，则省略前台步骤，直接在后台页面新增车辆）

### 7.3 后台录入交易流程

1. 后台管理人员登录本系统后台。
2. 进入“交易管理 → 新增交易”页面。
3. 输入用户关键字（姓名/手机号/个人番号卡）进行搜索，并选择正确的用户。
4. 选择回收日期、品类、重量、单价或金额、车辆信息、结算方式等。
5. 提交后，系统写入 `transactions` 表，并在 `operation_logs` 中记录操作。
6. 如有需要，生成内部单号或打印凭证。

### 7.4 查询历史交易流程

1. 后台管理人员打开“交易查询”页面。
2. 输入查询条件：
   - 关键字（姓名/手机/个人番号卡）
   - 回收品类
   - 时间条件（指定月份、年份或者起止日期）
   - 其它条件（如金额区间）
3. 点击查询，系统从 `transactions` 联表 `users`、`categories` 等表查询数据。
4. 返回列表，并在页面底部显示汇总信息（总重量、总金额等）。
5. 如需要，点击“导出”将当前查询结果导出为 Excel/CSV。

## 8. 界面设计（概要）

### 8.1 菜单结构建议

- 首页 / 仪表盘（可选）
- 用户管理
  - 用户列表
  - 车辆信息管理
- 交易管理
  - 新增交易
  - 交易列表 / 交易查询
- 品类管理
  - 品类列表
- 系统管理
  - 管理员账号
  - 角色与权限
  - 操作日志

### 8.2 重要页面说明

1. **用户列表页面**
   - 顶部搜索框：姓名、手机号、个人番号卡。
   - 列表：姓名 | 手机 | 个人番号卡 | 注册时间 | 状态 | 操作（查看详情）。
2. **新增交易页面**
   - 用户搜索框 + 选择结果。
   - 品类下拉框、重量输入框、单价与金额、车辆下拉框。
   - 备注多行文本。
3. **交易查询页面**
   - 查询条件区域：关键字、品类、多选时间（按月/按年）、金额区间。
   - 查询结果表格及汇总信息。
   - 导出按钮。

  - Vue / React / 或公司现有前端框架
  - UI 组件库如 Element-UI、Ant Design 等

  - Java（Spring Boot）或 Node.js（Express/NestJS）或 .NET Core 等

  - MySQL / PostgreSQL / SQL Server
- 部署：
  - Docker 容器化部署（可选）
  - Nginx 作为反向代理
- 接口方式：
  - RESTful API + JSON
